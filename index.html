Введение и Аутентификация

Добро пожаловать в полную документацию системы мониторинга локомотивов. Данное API позволяет как получать данные (телеметрию, координаты) в реальном времени, так и управлять инфраструктурой.

Авторизация обязательна: Все API эндпоинты защищены через Laravel Sanctum (auth:sanctum). При запросах необходимо передавать Bearer токен в заголовке авторизации, либо использовать сессии для SPA приложений: Cookie: XSRF-TOKEN.

Стандартные маршруты авторизации (Fortify)

  • POST /login — Аутентификация (email, password).
  • POST /logout — Завершение сессии.
  • POST /register — Регистрация нового оператора/пользователя.
  • POST /forgot-password — Запрос на сброс пароля.

API: Локомотивы

CRUD /api/locomotives[/{id}]

Базовые операции для работы с реестром локомотивов автопарка.

  • GET /api/locomotives — Список всех локомотивов в системе.
  • POST /api/locomotives — Регистрация нового локомотива (Требуется: id, model, Опционально: depot).
  • GET /api/locomotives/{id} — Получение полной информации об 1 локомотиве.
  • PATCH /api/locomotives/{id} — Обновление данных локомотива.
  • DELETE /api/locomotives/{id} — Удаление локомотива из базы данных (204 No Content).
GET /api/locomotives/{id}/stats
Телеметрия

Возвращает до 500 последних исторических записей телеметрии для конкретного локомотива в обратном хронологическом порядке (от самых новых к старым).

Пример ответа:

200 OK
[
  {
    "id": 8021,
    "locomotive_id": "VL80S-100",
    "timestamp": "2026-04-05T08:00:00Z",
    "speed": 85.2,
    "temperature": 90.5,
    "pressure": 110.2,
    "fuel_level": 75.0,
    "health_score": 98,
    "is_anomaly": false
  }
]

API: Телеметрия и Аналитика

GET /api/map/positions

Возвращает последние известные геопозиции и текущий статус по каждому локомотиву для отрисовки на интерактивной карте.

Ответ:

[
  {
    "id": "VL80S-100",
    "series": "VL80",
    "lat": 48.8566,
    "lng": 2.3522,
    "speed": 65,
    "status": "В пути"
  }
]
GET /api/telemetry/analytics

Агрегация либо сырые показания телеметрии локомотива за определенный временной промежуток.

Query Параметры:

  • locomotive_id (string, обязательно) - ID локомотива.
  • start_date (date, обязательно) - Начало выборки.
  • end_date (date, обязательно) - Конец выборки (должно быть больше start_date).
  • step (integer, опционально) - Шаг агрегации (в секундах). Если передан, данные усредняются функцией date_bin базы данных.
GET /api/telemetry/analytics/all

То же самое, но вычисляет усредненную аналитику по всему парку локомотивов (полезно для графиков Fleet-Wide Analytics).

  • Требует параметры start_date и end_date. По умолчанию step равен 300 (5 минут).
GET /api/telemetry/anomalies

Журнал всех зафиксированных аномальных показателей (перегрев, критическое давление) в системе.

  • limit (int, опционально) - Максимальное количество результатов. По умолчанию: 50. Упорядочено по времени (новые сначала).

API: Инфраструктура

CRUD /api/endpoints[/{id}]

Управление конфигурацией WebSocket эндпоинтов и каналов приема потоковых данных (создание, удаление, редактирование).

Параметры при POST/PATCH: url (строка, URL, уникальный), is_active (булево), description (текст).

GET /api/user

Возвращает профиль текущего пользователя по авторизационному токену / сессии.

Веб: Эндпоинты интерфейса (SPA / Inertia.js)

Данные маршуты отдают не JSON, а клиентский интерфейс Svelte через компонент инерции (Inertia.js). Для доступа пользователь должен быть залогинен (middleware 'auth').

WEB /dashboard

Главный экран с картой. Перенаправляет сюда с /. Компонент: Dashboard.svelte

WEB /analytics

Сводная аналитика по автопарку. Компонент: Analytics.svelte
