Introduction & Authentication

Welcome to the API Documentation for the Locomotive Fleet Telemetry Dashboard. This API allows fetching real-time and historical locomotive data, including metrics, positions, warnings, and websocket routing details.

Authentication Required: All API endpoints are protected by Laravel Sanctum (`auth:sanctum`). You must include a valid Bearer token or use cookie-based session authentication for SPA requests.

User Endpoints

GET /api/user

Returns the currently authenticated user details.

Responses

200 OK
{
  "id": 1,
  "name": "Operator",
  "email": "operator@locomotive.example.com",
  "created_at": "...",
  "updated_at": "..."
}

Locomotives

GET /api/locomotives

Retrieves a list of all locomotives in the fleet.

POST /api/locomotives

Creates a new locomotive record in the database.

Request Body

  • id (string, required): Unique identifier for the locomotive.
  • model (string, required): Model/Series. Max 255 chars.
  • depot (string, optional): Assigned depot. Max 255 chars.
GET /api/locomotives/{id}

Retrieves the details of a specific locomotive by its ID.

PUT/PATCH /api/locomotives/{id}

Updates an existing locomotive record.

Request Body

  • model (string, optional)
  • depot (string, optional)
DELETE /api/locomotives/{id}

Deletes a locomotive. Returns 204 No Content on success.

GET /api/locomotives/{id}/stats

Telemetry Analytics

GET /api/telemetry/analytics

Fetches aggregated or raw telemetry data for a specific locomotive across a given time window.

Query Parameters

  • locomotive_id (string, required)
  • start_date (date, required)
  • end_date (date, required, after start_date)
  • step (integer, optional) - If provided, records are grouped directly via date_bin using the step size in seconds.
GET /api/telemetry/analytics/all

Calculates fleet-wide averages for speed, temperature, pressure, and fuel level across the provided time window.

Query Parameters

  • start_date (date, required)
  • end_date (date, required, after start_date)
  • step (integer, optional) - Data grouping interval in seconds. Defaults to 300 (5 minutes).
GET /api/telemetry/anomalies

Retrieves the historical list of acknowledged and recent telemetry anomalies detected across the fleet.

Query Parameters

  • limit (integer, optional) - Max objects to return. Default 50.

Websocket Endpoints

REST /api/endpoints

Standard CRUD resource for managing Websocket Endpoints mapped for data ingestion, connection handlers or real-time broadcast receivers.

  • GET /api/endpoints - List all endpoints
  • POST /api/endpoints - Store new endpoint (parameters: url, is_active, description)
  • GET /api/endpoints/{id} - Read details of specific endpoint configuration
  • PUT/PATCH /api/endpoints/{id} - Update an existing endpoint
  • DELETE /api/endpoints/{id} - Delete endpoint
