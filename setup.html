Как развернуть проект (Getting Started)

В этой инструкции подробно описано, как клонировать репозиторий, установить зависимости и запустить проект локально или на сервере. Проект использует стек Laravel (PHP), Inertia.js и Svelte.

Шаг 1. Клонирование и зависимости

Сначала скачайте код и установите пакеты (PHP и Node.js).

git clone <ваш-репозиторий> locomotive-app
cd locomotive-app

# Установка PHP-зависимостей (composer)
composer install

# Установка JS-зависимостей (NPM)
npm install

Шаг 2. Конфигурация окружения (.env)

Скопируйте пример конфига и установите ключ:

cp .env.example .env
php artisan key:generate

Откройте файл .env и настройте конфигурацию БД. Например, для PostgreSQL: 

DB_CONNECTION=pgsql
DB_HOST=127.0.0.1
DB_PORT=5432
DB_DATABASE=locomotive
DB_USERNAME=postgres
DB_PASSWORD=secret

Шаг 3. Запуск миграций и сидов

Создайте таблицы в базе данных и заполните их демо-данными (в т.ч. тестовыми локомотивами):

php artisan migrate --seed

Шаг 4. Запуск серверов

Так как мы используем Vite + Svelte для интерфейса, нам понадобятся два процесса (в разных терминалах):

# 1-й терминал (Backend)
php artisan serve

# 2-й терминал (Frontend сборщик)
npm run dev

Успех: Приложение будет доступно по адресу http://127.0.0.1:8000. Вы можете войти с учетными данными, сгенерированными сидами!

Дополнительно: Запуск веб-сокетов (Reverb)

Если вы используете систему телеметрии в реальном времени, вам нужно поднять веб-сокет сервер:

php artisan reverb:start
php artisan queue:work
