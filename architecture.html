Как всё устроено (Архитектура)

Система мониторинга локомотивов спроектирована с использованием современных технологий и условно делится на Backend (работа с БД, API, бизнес-логика) и Frontend (визуализация, карты, SPA).

Технологический Стек

  • Laravel 11 / PHP 8.3+: Ядро платформы. Обеспечивает роутинг (API + Web), ORM (Eloquent), аутентификацию (Fortify) и очереди.
  • Inertia.js + Svelte 5: Позволяет нам писать современный Single-Page Application (SPA) интерфейс без сложной конфигурации клиентского роутинга. Контроллеры Laravel отдают готовые Svelte-компоненты.
  • Tailwind CSS 3/4: Вспомогательный CSS фреймворк для быстрого и красивого дизайна утилитными классами.
  • PostgreSQL: Реляционная БД. Идеально справляется с агрегацией таймсерий (данных телеметрии). Оптимизирована с помощью индексов и date_bin агрегаций.
  • Laravel Reverb (или Pusher): Движок WebSocket для отстреливания событий в реальном времени (например, изменение статуса локомотива).

Архитектура Базы Данных

Ключевые сущности системы:

users
Операторы системы. Управляют доступом и конфигурациями.
locomotives
Реестр самих локомотивов: id (номер), model (серия), текущее состояние и последние координаты.
telemetry_histories
Таблица, куда записывается гигантский поток показаний сенсоров: скорость (speed), температура (temperature), давление, уровень топлива. Имеет индексы по timestamp для быстрых фильтров аналитики.
websocket_endpoints
Адреса, по которым распределяются потоки данных локомотивов.

Жизненный цикл Телеметрии

  1. Датчики локомотива каждые N секунд генерируют пакет с данными (gps координаты, скорость, нагрев двигателей).
  2. Приемник / API Сервер получает этот запрос, валидирует (через API контроллеры в Laravel) и сохраняет в БД (таблице telemetry_histories).
  3. Если данные выходят за пределы нормы, срабатывает система Anomalies (запись флага is_anomaly=true).
  4. Сервер через WebSocket (Broadcast) мгновенно уведомляет всех подключенных диспетчеров (на клиентской стороне обновляется карта и счетчики без перезагрузки страницы).
  5. Диспетчер может открыть страницу Analytics, где данные извлекаются, группируются через функцию date_bin и отдаются в виде красивых графиков в интерфейсе Svelte.

Модуль Wayfinder: Обратите внимание, что маршрутизация на фронтенде работает "из коробки" благодаря плагину wayfinder(), который позволяет вызывать маршруты Laravel напрямую в JS (через `wayfinder.route(...)`).
